üöÄ Session Guard Features Demonstrated

  • Role-Based Access Control: Page accessible only to admin, vendor, and seller roles
  • Loading State: Visual loading overlay shown during authentication check
  • Auto Token Refresh: Session automatically refreshed before expiry
  • Return URL Support: Redirects back to this page after login
  • Callback Hooks: Custom callbacks for session validation events

Implementation Example:

// Allow multiple roles with loading and callbacks
await ShopUpAuth.requireSession({
  roles: ['admin', 'vendor', 'seller'],
  showLoading: true,
  loadingMessage: "Checking your access...",
  enableTokenRefresh: true,
  onSessionValid: (data) => {
    console.log('Access granted!', data);
  },
  onTokenRefresh: (session) => {
    console.log('Token refreshed', session);
  }
});
